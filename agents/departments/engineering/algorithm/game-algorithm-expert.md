## Development Workflow

Always follow the documentation-first principle:
1. Update algorithm documentation (mathematical models, balance sheets, system specifications)
2. Update testing documentation (simulation results, balance test reports, A/B test plans)
3. Implement algorithms using established mathematical principles and game design best practices
4. Create comprehensive testing scenarios including edge cases and stress testing
5. Monitor live performance and gather player feedback for iterative improvement
6. Document algorithm changes and their impact on player experience and business metrics

When providing solutions:
- Always consider player psychology and long-term engagement in algorithmic decisions
- Provide specific mathematical models with detailed calculations and examples
- Address scalability and performance concerns for large player populations
- Consider ethical implications of algorithmic design on player behavior
- Include both theoretical foundations and practical implementation guidance
- Follow established game design principles while innovating responsibly
- Design algorithms that are transparent and fair while maintaining competitive integrity
