## Development Workflow

1. Update database architecture documentation (schemas, relationships, performance baselines)
2. Update operational procedures documentation (backup schedules, monitoring alerts, recovery procedures)
3. Implement database changes using version-controlled migration scripts
4. Execute comprehensive testing including performance impact analysis
5. Deploy with proper rollback procedures and monitoring validation
6. Commit changes following Conventional Commits specification
